GenCyberCoin has a Bug Bounty program! This program is a type of deal that is offered by websites, organizations, and even developers by which individuals will be compensated for finding bugs (errors, exploits, vulnerabilities) on websites or programs.

Bug Bounty hunting is a real-world practice where people can receive monetary rewards on websites including HackerOne and BugCrowd. It can be quite profitable to participate in these “ethical hacking” Bug Bounty program events. A 10-year-old was reward $10,000 by Facebook in 2016 for finding a serious bug on Instagram! You can read more about that story here: Click here.

Try to do something that the website would not expect a normal user to do. Finding a bug will present a special message, and you will be given REWARD coins! :D

Do note that some bugs will present codes that you will need to redeem in order to receive any REWARD coins; head to the Wallet page to redeem them!

Hint: Right-click on a specific item on any page and select “Inspect” to see the underlying code.

BEFORE performing a cyberattack, ALWAYS think about ethics first. If you were to perform an unethical cyberattack, heavy charges including FINES and PRISON TIME can be placed upon you!

For this website though, you can try to break it in any way you want!

  1. Sensitive data exposure (hidden page that admin left somewhere on the home page's source code)
  2. Broken authentication (find admin credentials)
  3. Find a proposed standard which allows websites to define security policies (search this in the Internet and you shall stand on the path to find the answer to this one)
  4. User data input overflow (check out maxlength in page sources)
  5. Reflected XSS (check out wallet)
  6. Stored XSS (btw, did you leave us the feedback yet?)
  7. User input validation (wallet)
  8. HTML edits on social engineering page (how does the server know what question relate to what answer?)
  9. Have you thought what happens if you transfer coins to yourself? (find in the wallet's page source the numerical values that correspond to users, replace one of them with yours: either guess it or the Blockchain page may help you)
  10. Local file inclusion (did you see that there are several pages of the items on the market?)
  11. HTML edits on market items (can you change the price of the items so that the money will actually come to you when you buy, instead of going out of your pocket?)
  12. Race condition (when market is enabled, notice what changes in every item's HTML code when it is your turn; then, when it is not your turn, can you still try to buy something?)
  13. Refresh on remove (when market is on)
  14. Access a page that does not exist
  15. Break the hash of wonderwoman and log in to her account (you will find it when you solve bug#2)
  16. Find a forgotten form on Cryptocurrency page and use it to unlock Atlantis (check out the page source; inspect the elements on the page and look for a form which display is none; find out how to make that form visible)
  17. Cause a CSRF failure (learn about CSRF; notice that the forms, where you can submit something, always have some kind of hidden field in it; what happens when you delete it and submit)

Note: If you see a page with a yellow background with random code, this means that you have found an actual bug. Please report this bug to Vitaly Ford (fordv@arcadia.edu ) with an attached screenshot. Depending on the severity of the bug, you may be rewarded outside of this camp/class.
